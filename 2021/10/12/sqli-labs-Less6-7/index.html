2021-10-12

再接再厉，本期博客记录着我们又学到新知识和新工具，以下是关于Less6~7的相关内容。

Less6

Less6与Less5同属于Bool型盲注，布尔盲注一般适用于页面没有回显字段(不支持联合查询)，且web页面返回True 或者 false，构造SQL语句，利用and，or等关键字来其后的语句 true 、 false使web页面返回true或者false，从而达到注入的目的来获取信息。

Less5已经对所需使用的函数有了大致的解释，我在这里做了一个简单的整理。

1、left()函数：select left(database(),1)=‘’； // left(a,b)从左侧截取a的前b位，正确则返回1，错误则返回会0

2、regexp函数：select user() regexp ‘r’； // user()的结果是root，regexp为匹配root的正确表达式

3、like函数：select user() like ‘r%’； // 匹配与regexp相似，区别是后面加个%

4、substr(a,b,c) select substr(a,b,c)=‘’； substr(a,b,c)从位置b开始，截取a字符串c位长度

5、ascii select ascii (‘’)； // 将某个字符串转化为ascii值

6、chr(数字) 或者是ord(‘字母’)； // 使用python中的两个函数可以判断当前的ascil值是多少

7、length(database())=‘’ // 判断数据库字符串长度

8、limit x,y 查询指定的数据库，其中x和y都表示数字，意思是查询从x开始y个数据库，起始为0.

Less7

在开始Less7的内容之前，我们再一次来理解SQL注入的内涵。

首先，这里有一个小技巧，我们可以通过测试?id='来判断是否存在SQL注入。如果提示You have an error in your SQL syntax，那么就说明存在SQL注入。

1、测试注入点，对id语句进行猜测：

单引号测试： id?=1’–+ 数据库报错
双引号测试： id=?1" --+ 页面正常显示
通过这两条信息的对比，单引号是闭合条件之一

带着单引号 ’ 继续猜测
单引号+括号测试： ?id=1’) --+ 数据库报错
单引号+双括号测试： ?id=1’)) --+ 页面正常显示
这里我们测试出了正确的闭合语句，
可以推断出后端的SQL语句大概是 select * from users where id = ((‘$id’))
(注释：这里直接猜测 单引号后面加上括号，是由于php脚本语言，能识别的解析变量的字比较少，主要就是 单引号、双引号、括号等)

2、into outfile

我们注入正确后，会看到画面出现如下：

这就是导出文件字符型注入，所以这是要就很有必要先学习文件字符

into outfile 写文件
用法：select’内容’into outfile ‘位置’; // 记得\

用法：select load_file(‘位置’);
